Ответы на вопросы пр3

1.Разница между селектором по классу и по ID
ОТВЕТ: Класс (.class): специфичность = 0-1-0. Можно использовать много раз на одной странице для одинакового оформления. Переиспользуемый селектор.ID (#id): специфичность = 1-0-0. Должен быть уникален на странице (теоретически только один элемент). Используется для уникальных, единственных элементов.
Класс лучше использовать для повторяющихся стилей (кнопки, карточки, выделения), ID — для уникальных элементов (заголовок страницы, основной контейнер)

2. Как работает каскад в CSS и порядок применения стилей
ОТВЕТ: Каскад — последовательное применение CSS-правил, где последнее правило перезаписывает предыдущие. Порядок применения при конфликте:
1) Селекторы элементов (низкая специфичность: 0-0-1)
2) Селекторы классов (средняя специфичность: 0-1-0)
3) Селекторы ID (высокая специфичность: 1-0-0)
4) Инлайн-стили (очень высокая: 1-0-0-1)
5) !important (максимальная приоритетность)

3. Расчет специфичности селекторов
ОТВЕТ:ul li.important
- 1 элемент (ul) = 0-0-1
- 1 элемент (li) = 0-0-1
- 1 класс (.important) = 0-1-0
Итого: 0-2-1
#site-title
- 1 ID = 1-0-0
Итого: 1-0-0
.content > .featured-post
- 1 класс (.content) = 0-1-0
- 1 класс (.featured-post) = 0-1-0
Итого: 0-2-0

4.Наследуемые и ненаследуемые свойства
ОТВЕТ:Наследуются (передаются от родителя к потомкам):
- color (цвет текста)
- font-family (шрифт)
- line-height (высота строки)
- font-size (размер шрифта)
- font-weight (толщина шрифта)
- text-align (выравнивание текста)
НЕ наследуются:
- width и height (размеры элемента)
- margin и padding (внешние и внутренние отступы)
- border (граница элемента)
- background (фон элемента)
- position и top/left/right/bottom (позиционирование)
- display (способ отображения)

5.Селектор > (дочерний) и селектор потомков (пробел)
ОТВЕТ:Селектор дочерних элементов (>): main > article — выбирает только article, которые являются ПРЯМЫМИ дочерними элементами main (на первом уровне вложенности)
Селектор потомков (пробел): article p — выбирает все p, которые находятся ВНУТРИ article на любом уровне вложенности (могут быть глубоко вложены)
Дочерний селектор более строгий и специфичный, селектор потомков более гибкий

6.Псевдоклассы :hover и :nth-child()
ОТВЕТ:hover — активируется при наведении мыши на элемент. Используется для интерактивности: подсветка кнопок, ссылок, изменение цвета при наведении.
:nth-child() — выбирает элементы по их порядковому номеру среди соседей. Используется для чередования цветов в таблицах (odd/even), отдельного оформления первого или последнего элемента в списке.

7 Правило !important и осторожность при использовании
ОТВЕТ:!important повышает специфичность селектора до максимума и перезаписывает все остальные стили, включая инлайн-стили.
Почему нужно использовать с  осторожностью:
- Нарушает логику каскада CSS
- Усложняет отладку и поддержку кода
- Если нужно переопределить !important, придётся использовать !important снова
- Делает код хрупким и неустойчивым к изменениям
- Плохая практика для команды разработчиков

8. Псевдоэлементы ::before и ::after
ОТВЕТ::before — добавляет контент ДО элемента (перед ним).
::after — добавляет контент ПОСЛЕ элемента (позади него).
Могут содержать:
- Текстовый контент через content: "блаблабла"
- Иконки и символы через content
- Пустой контент для создания декоративных слоёв
- Могут быть стилизованы отдельно (цвет, размер, позиция)
Расширяют возможности: позволяют добавлять визуальные элементы без изменения HTML.

9. Порядок применения CSS-правил при одинаковой специфичности
ОТВЕТ:Браузер применяет последний стиль из CSS-файла. Если есть два правила с одинаковой специфичностью, побеждает то, которое определено ниже в коде.
Пример:
p { color: blue; }  /* Сначала синий... */
p { color: red; }   /* ...но потом переопределится на красный */

10. Инструменты разработчика браузера для CSS
ОТВЕТ:
DevTools браузера позволяют:
- Видеть все применённые и переопределённые стили для каждого элемента
- Проверять специфичность каждого CSS-селектора
- Отключать отдельные стили и видеть результат в реальном времени (без перезагрузки)
- Анализировать бокс-модель (margin, padding, border, content)
- Находить переопределённые стили (отображаются зачёркнутыми)
- Изменять значения на лету и тестировать без сохранения файла
- Проверять наследование свойств от родительских элементов
- Инспектировать структуру DOM и видеть классы/ID элементов
