Ответы на вопросы
1. Какие три основные области работы с файлами в Git и чем они отличаются?
ОТВЕТ: Работа с файлами в трёх местах: в рабочей папке (где делаются изменения), в области подготовки (куда добавляются изменения командой git add), и в репозитории (где хранится вся история коммитов); 

2. Что происходит при выполнении команд: git add, git commit, git push? 
ОТВЕТ: 1) Когда делается git add, выбираются файлы, которые хочу добавить к следующему коммиту. 2)git commit сохраняет эти изменения в локальной истории. 3)git push отправляет коммиты на удалённый сервер, чтобы я мог поделиться ими или сохранить их в облаке; 

3. Почему важно писать информативные сообщения к коммитам? 
ОТВЕТ: Чтобы я и другие могли понять, что именно изменили и зачем, когда будут просматривать историю изменений; 

4. Чем отличается git add имя_файла от git add . и когда их использовать? 
ОТВЕТ: 1)git add имя_файла добавляет только один конкретный файл 2) git add . добавляет все изменения в текущей папке; 

5.Что показывает git status и как меняется его вывод? 
ОТВЕТ: Он показывает мне, какие файлы изменены, какие подготовлены к коммиту, и какая ветка сейчас активна. Его вывод меняется, когда делаются git add, git commit или другие команды; 

6.Для чего нужна команда git remote add origin URL и что значит origin? 
ОТВЕТ: связывается свой локальный репозиторий с удалённым хранилищем на сервере, чтобы потом отправлять туда свои изменения. origin — это стандартное название этого удалённого места; 

7. Почему в команде git push -u origin main есть флаг -u? 
ОТВЕТ: Он говорит Git запомнить привязку моего локального ветки к ветке на сервере, чтобы в следующий раз я мог просто писать git push; 

8. Что такое ветка по умолчанию и какая у меня — main или master? Почему это важно? 
ОТВЕТ: Это основная ветка проекта, обычно она называется main сейчас или master раньше. Это важно, потому что все основные изменения обычно делаются именно в ней;

9. Как я могу увидеть историю своих изменений? 
ОТВЕТ: Используя команду git log, которая показывает список всех коммитов с информацией о авторе, дате и сообщении; 

10.Что произойдет, если я сделаю изменения в файле, но не выполню git add, перед git commit? 
ОТВЕТ: Изменения останутся только в рабочей папке и не попадут в историю, то есть не будут зафиксированы в коммите, пока я не добавлю их в git add
