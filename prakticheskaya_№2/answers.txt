Ответы на вопросы по практической работе №2

1. Чем семантическая верстка отличается от несемантической и почему она важна?
ОТВЕТ: Семантическая верстка использует теги с смысловым значением (<header>, <article>, <nav>), несемантическая — нейтральные <div> и <span>. Это важно для доступности, SEO, читаемости кода и удобства разработки

2. В каких случаях следует использовать <article> вместо <section>?
ОТВЕТ: <article> — для независимого контента (может быть синдицирован), <section> — для тематической группировки. В работе: <article> для статей об истории, архитектуре, замках; <section> для подразделов внутри статей

3. Для чего предназначены теги <header> и <footer> и могут ли они использоваться несколько раз?
ОТВЕТ:<header> — вводная часть (заголовок, навигация), <footer> — заключительная (копирайт, ссылки). Да, могут использоваться несколько раз: глобальные для всей страницы и локальные внутри статей

4. Какой тег для основного контента и почему не <div>?
ОТВЕТ:<main> — явно указывает на основной контент, программы чтения с экрана могут переходить напрямую к нему, поисковики лучше индексируют. <div> — просто контейнер без значения

5. Разница между <nav> и обычным списком ссылок?
ОТВЕТ: <nav> — семантический тег, обозначает навигацию. Используется для главного меню, внутристраничной навигации, breadcrumbs. Обычный список ссылок значения не имеет

6. Для чего атрибут datetime в теге <time>?
ОТВЕТ: Атрибут datetime в теге <time> используется для предоставления машиночитаемой версии даты и времени, которая не отображается напрямую в браузере

7. Преимущества <figure> и <figcaption> вместо простого <img>?
ОТВЕТ: Использование <figure> и <figcaption> вместо простого <img> улучшает семантику, доступность и организацию контента, позволяя связывать изображение с его подписью, а не просто отображать картинку

8. Когда использовать <aside>?
ОТВЕТ: <aside> нужен для вспомогательного контента, связанного с основным, но не являющегося его частью: боковые панели, популярные статьи, интересные факты. В работе — список популярных статей и факты о замках

9. Влияние семантической разметки на SEO и доступность?
ОТВЕТ: SEO:лучше индексируется структура, поисковики распознают основной контент, улучшается ранжирование. Доступность:программы чтения быстро навигируют по разделам, пользователи с нарушениями находят информацию легче

10. Какие теги использовали для улучшения структуры?
<header> — шапка страницы и статей
<nav> — навигационное меню
<main> — основной контент
<article> — статьи об истории, архитектуре, замках
<section> — подразделы внутри статей
<aside> — боковая панель с дополнительной информацией
<footer> — подвал страницы
<time datetime=""> — даты публикации
<figure>/<figcaption> — изображения с подписями
<table> — таблицы элементов замков и хронология
<form> — форма подписки
